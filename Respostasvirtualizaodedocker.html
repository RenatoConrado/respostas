20 – Descrever o significado, impacto e a remediação de um requisito de segurança que teve como resultado [PASS] e outro que teve como resultado [WARN] para a análise de “Host Configuration”

[PASS] 1.3 - Ensure Docker is up to date

Significado:

Significa que passou na verificação de que a versão instalada do Docker está atualizada.

Impacto:

Manter o Docker atualizado protege contra brechas na segurança e bugs. Utilizar uma versão desatualizada expõe o sistema a ataques que já foram corrigidos.

Remediação:

Se manter sempre atualizado, com por exemplo “sudo apt update”

[WARN] 1.1 - Ensure a separate partition for containers has been created

Significado:

Avisa que não passou na verificação de criar uma partição separada para o diretório “/var/lib/docker”.

Impacto:

Se o diretório “/var/lib/docker” compartilhar espaço com outras partes críticas do sistema, como /, o aumento dos dados dos containers pode acabar com o espaço do disco.

Remediação:

Criar uma partição separada para o diretório “/var/lib/docker”.

exemplo:

“sudo rsync -aP /var/lib/docker /nova-particao/docker”

“sudo mount”/nova-particao/docker /var/lib/docker”

21 – Descrever o significado, impacto e a remediação de um requisito de segurança que teve como resultado [PASS] e outro que teve como resultado [WARN] para a análise de “Container Images and Build File”..

[PASS] 4.7 - Ensure update instructions are not used alone in the Dockerfile

Significado:

Significa que passou no requisito que garante que comandos como “apt-get update” não estejam sendo usados no Dockerfile.

Impacto:

Usar comandos como “apt-get update” sozinho pode criar uma imagem onde as fontes de pacotes mudam entre builds, comprometendo a segurança ou funcionalidade.

Remediação:

Combinar os comandos “update” e”install” em uma linha só de comando para evitar problemas: “RUN apt-get update && apt-get install -y pacote”

[WARN] 4.6 - Ensure HEALTHCHECK instructions have been added to the container image

Significado:

Signfica que não passou no requisito que verifica se as imagens possuem instruções HEALTHCHECK, que monitoram a saúde do container e notificam automaticamente falhas.

Impacto:

A ausência de “HEALTHCHECK” dificulta a identificação de problemas, fazendo os containers rodarem sem ninguém saber se tem problemas ou não.

Remediação:

Adicionar uma instrução “HEALTHCHECK” no Dockerfile da imagem para verificar sua funcionalidade.

Por exemplo:

HEALTHCHECK --interval=30s --timeout=5s \

  CMD curl -f http://localhost/ || exit 1
